# 售后页面样式与流程截图

用于沉淀售后相关页面的样式和操作流程截图，方便其他开发者快速查看当前 UI 与交互状态。

## 建议放置方式

- 原图直接放在当前目录，优先保留 PNG 格式。
- 文件名保持稳定，后续如有替换，尽量覆盖同名文件，避免文档链接失效。
- 若截图有更新，请同步修改下方说明，标注变化点。

## 当前截图清单

1. `01-submit-consumer-negotiation-plan.png`
   - 内容：请与消费者协商方案弹窗，展示第一轮售后协商的步骤与选项样式。
   - 用途：给开发、测试和产品确认第一轮协商弹窗的视觉与文案。

2. `02-second-round-negotiation-dialog.png`
   - 内容：桌面端页面中的第二轮售后协商弹窗，包含责任原因勾选、推荐话术与提交区。
   - 用途：用于对照完整页面上下文中的弹窗布局和流程位置。

3. `03-reject-refund-step-one.png`
   - 内容：驳回退款弹窗第一步，包含驳回原因、是否协商售后方案以及可选方案区域。
   - 用途：用于确认驳回退款流程第一步的交互和选项展示。
